# Product Constitution Entry Point

This repository follows a formal product constitution and mandatory change guardrails.

Primary references:
- `docs/PRODUCT_CONSTITUTION.md`
- `docs/CHANGE_GUARDRAILS.md`
- `design_guidelines.md`
- `docs/VISUAL_QA_MATRIX.md`

If any implementation detail conflicts with old assumptions, use these documents as the current governing baseline.

UI non-negotiable:
- Keep a unified design system across all pages and components.
- Do not introduce ad-hoc spacing/typography/color patterns outside the documented tokens and primitives.
- Any UI change must be validated for mobile, tablet, and laptop, with Arabic and English parity.
